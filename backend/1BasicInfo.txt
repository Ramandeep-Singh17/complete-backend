BACKEND PRACTICE NOTES

Sabse pahle:
HM HAMESHA FULLSTACK PROJECT ME PAHLE BACKEND BANATE HAI

---

Core Concept

Hum database me image ko directly store nahi karte
Hum image ko cloud storage me store karte hain aur uska URL database me save karte hain

Kyuki:
Image heavy hoti hai aur database slow ho jata hai
URL lightweight hota hai aur fast access milta hai

Humne cloud storage ke liye ImageKit use kiya

---

.env ka use

.env file me hum sensitive data store karte hain:

* MongoDB connection string
* ImageKit private key
* public key
* url endpoint

Example:
MONGODB_URI=...
IMAGEKIT_PRIVATE_KEY=...
IMAGEKIT_PUBLIC_KEY=...
IMAGEKIT_URL_ENDPOINT=...

Code me use:
process.env.MONGODB_URI

Kyuki:

* security maintain hoti hai
* keys expose nahi hoti
* easily change kar sakte hain

---

SRC Folder kya hai aur kyu use karte hain

src (source) folder me hum apna main code rakhte hain

Kyuki:

* project clean rehta hai
* sab logic ek jagah hota hai
* industry standard structure hai

Iske andar:

* app.js (main app logic)
* db (database connection)
* models (schema & models)
* services (external services jaise ImageKit)

---

Basic Steps of Backend

1. Project Setup
   node init kiya
   express aur mongoose install kiya

2. Server Creation
   src/app.js me express app banaya
   server.js me app import karke server start kiya

3. Database Connection
   src/db/db.js me MongoDB connect kiya using .env

4. Schema
   data ka structure define kiya
   ise schema bolte hain

5. Model
   src/models/post.model.js me schema banaya
   data create aur fetch kiya

6. Services Folder
   src/services me ImageKit logic rakha

Kyuki:
cloud service change ho sakti hai
isliye isolate rakha

7. Multer
   file handle karne ke liye multer use kiya

Kyuki:
express direct file handle nahi kar pata
multer req.file provide karta hai

8. API Creation

POST API (/create-post):

* Postman se image + caption aaya
* Multer ne file handle ki
* buffer ko base64 me convert kiya
* ImageKit pe upload kiya
* URL mila
* MongoDB me save kiya

GET API:

* database se data fetch kiya

---

FLOW (Actual Implementation)

.env load (DB URI + ImageKit keys)
↓
Server start

↓

↓

↓

↓
Response
